# Before / after gallery

This page captures the kind of improvement design skills should create.

## Example 1: generic SaaS dashboard hero

### Prompt used

Create a modern SaaS analytics dashboard hero with KPI cards, charts, and a CTA section.

### Tool and guidance applied

- **Tool:** Claude Code-style frontend workflow
- **Primary skill:** Anthropic frontend-design skill
- **Evaluation lens:** Anti-slop rubric

### Before

- weak gray-on-gray hierarchy
- interchangeable blue gradient accents
- one large card grid with little priority
- no interaction detail

### After

- clearer information hierarchy
- stronger typography contrast
- more deliberate spacing rhythm
- distinct primary action and supporting actions
- purposeful motion in hover and panel transitions

### Rubric shift

- **Anti-slop rubric:** 3/8 strong areas → 7/8 strong areas

### Reusable takeaway

A strong frontend design skill should turn a generic dashboard hero into a screen with real hierarchy, product-specific emphasis, and more intentional interaction detail.

## Example 2: brandless landing page

### Prompt used

Design a landing page for a developer tool that feels confident, differentiated, and ready for launch.

### Tool and guidance applied

- **Tool:** Cross-tool DESIGN.md workflow
- **Primary skill:** VoltAgent/awesome-claude-design
- **Evaluation lens:** AI UI quality rubric

### Before

- generic startup copy blocks
- standard pill buttons and placeholder illustrations
- no memorable tone or visual system

### After

- recognizable visual voice
- clearer component rules
- stronger color logic tied to brand intent
- more product-specific section structure

### Rubric shift

- **AI UI quality rubric:** 19/35 → 29/35

### Reusable takeaway

Brand-system-oriented skills are most valuable when they create visual rules that can carry beyond a single page into a repeatable interface language.

## Example 3: cluttered app settings page

### Prompt used

Refine a dense settings screen for a team product so it feels clearer, safer, and more trustworthy.

### Tool and guidance applied

- **Tool:** Cursor or Codex refinement workflow
- **Primary skill:** design audit-oriented skill set
- **Evaluation lens:** Anti-slop rubric plus AI UI quality rubric

### Before

- dense forms with no grouping
- weak label hierarchy
- equal emphasis on destructive and safe actions

### After

- grouped controls by task
- better type scale and spacing
- clearer affordances and action priority
- improved empty, success, and error states

### Rubric shift

- **Anti-slop rubric:** 2/8 strong areas → 6/8 strong areas
- **AI UI quality rubric:** 17/35 → 27/35

### Reusable takeaway

Audit skills should not only point out what is wrong. They should help convert clutter into safer grouping, clearer hierarchy, and more trustworthy interaction states.
