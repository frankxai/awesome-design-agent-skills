# Before / after gallery

This page captures the kind of improvement design skills should create.

## Example 1: generic SaaS dashboard hero

### Before

- weak gray-on-gray hierarchy
- interchangeable blue gradient accents
- one large card grid with little priority
- no interaction detail

### After

- clearer information hierarchy
- stronger typography contrast
- more deliberate spacing rhythm
- distinct primary action and supporting actions
- purposeful motion in hover and panel transitions

## Example 2: brandless landing page

### Before

- generic startup copy blocks
- standard pill buttons and placeholder illustrations
- no memorable tone or visual system

### After

- recognizable visual voice
- clearer component rules
- stronger color logic tied to brand intent
- more product-specific section structure

## Example 3: cluttered app settings page

### Before

- dense forms with no grouping
- weak label hierarchy
- equal emphasis on destructive and safe actions

### After

- grouped controls by task
- better type scale and spacing
- clearer affordances and action priority
- improved empty, success, and error states

## What this gallery should evolve into

Eventually this page should include real screenshots, prompts, skill files, and notes on which design skill caused each improvement.
