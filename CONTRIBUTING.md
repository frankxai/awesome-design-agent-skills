# Contributing

This repository is curated, not exhaustive. The goal is to collect the highest-signal design skills and resources for AI coding agents, not every loosely related link.

## What counts as a design skill

A submission should directly improve one or more of the following:

- frontend UI generation quality
- motion and interaction polish
- brand system coherence
- design critique or audit quality
- reusable design-system prompting or skill structure

Resources that are only generic coding helpers, unrelated prompt packs, or broad agent directories without a design angle should usually not be added unless they clearly deserve inclusion as source pools.

## How rankings should work

Rankings should not be based primarily on stars, recency, or hype.

Rankings should favor resources that:

1. improve real output quality
2. reduce generic AI aesthetics
3. encode taste, hierarchy, spacing, typography, and interaction guidance
4. help at the system level, not only on one hero section or landing page
5. are reusable in real coding-agent workflows

## Evaluation process

Before proposing a new ranking entry or major reordering:

1. Read the relevant ranking page
2. Score the resource against the [Anti-slop rubric](./rubrics/anti-slop-rubric.md)
3. Score the resource against the [AI UI quality rubric](./rubrics/ai-ui-quality-rubric.md)
4. Explain why it belongs in this repo instead of a general skill directory
5. Compare it against the current items already listed in that category

## Submission checklist

Include the following in your issue or pull request:

- resource name and link
- which ecosystem it serves (Claude Code, Codex, Cursor, Gemini CLI, v0, Lovable, or cross-tool)
- which category it belongs to
- short description of what it actually helps with
- evidence of design usefulness, not just popularity
- rubric-based reasoning for why it should be added or ranked highly

## Suggested submission template

```md
- Resource: [name](url)
- Category: frontend / motion / brand system / audit / other
- Ecosystem: Claude Code / Codex / Cursor / Gemini CLI / v0 / Lovable / cross-tool
- Why it matters:
- Anti-slop rubric summary:
- AI UI quality rubric summary:
- Why it belongs here instead of a broader awesome list:
```

## Conflict of interest

If you are submitting your own project, say so clearly. Self-submissions are allowed, but they should be judged by the same bar as everything else.

## Editorial standard

Prefer fewer, stronger entries over long lists. Every page should help a reader make a faster, better decision.
