# Anti-slop rubric

Use this rubric to judge whether a design skill helps an agent escape generic AI-generated UI.

## 1. Visual identity

- **Strong:** output has a clear point of view and does not feel template-derived
- **Weak:** output looks like a default dashboard or landing page with interchangeable branding

## 2. Hierarchy

- **Strong:** headline, subhead, content, and actions have obvious priority
- **Weak:** everything competes for attention equally

## 3. Typography

- **Strong:** type scale, weight, and rhythm feel intentional
- **Weak:** typography is flat, uniform, or purely utility-driven

## 4. Spacing and layout

- **Strong:** spacing creates structure and confidence
- **Weak:** spacing feels arbitrary, crowded, or overly padded everywhere

## 5. Color and contrast

- **Strong:** color choices support meaning and brand feel
- **Weak:** output relies on generic blue-purple gradients or low-contrast gray palettes

## 6. Interaction polish

- **Strong:** hover, focus, loading, empty, and transition states feel considered
- **Weak:** only the static default state looks acceptable

## 7. System thinking

- **Strong:** components feel like part of one family
- **Weak:** each section feels generated independently

## 8. Product specificity

- **Strong:** the interface matches the product category and user intent
- **Weak:** the same layout could be used for any startup in any domain

## Quick scoring

- **7-8 strong areas:** excellent anti-slop skill
- **5-6 strong areas:** useful but inconsistent
- **3-4 strong areas:** average
- **0-2 strong areas:** mostly generic output
